Foundation

Core Principles

Five architectural decisions that shape everything in FORGE.

✍

Tickets Are Prompts

Every ticket is an agent-optimized context bundle. The full payload contains pre-assembled evidence, explicit instructions, success criteria, and constraints. Humans see only a thin summary on the board.

⚙

Three Agent Tiers

Gatherers (cheap, fast) collect and structure data. Thinkers (powerful) analyze and decide. Communicators (empathetic) translate outcomes into human language. Each uses the right model for the job.

⇄

Module Independence

All modules communicate through ticket contracts on the Board API. Any module can be swapped for a third-party tool. Modules never call each other directly. Bring your own IDE, support tool, or CI/CD.

⚖

Built on AIDEN

FORGE inherits LLM routing, RAG, queues, billing, auth, multi-tenancy, context engineering, and DAG execution from the proven AIDEN platform. ~35-50 weeks of engineering saved.

★

ROI Drives Everything

Every feature, bug fix, and task is scored by ROI. Agents default to minimal fixes and maximum pragmatism. They are economists, not perfectionists. The board is a prioritization tool.

Agent Architecture

Three-Tier Agent Model

Different cognitive jobs need different models. Splitting saves cost, improves quality, and makes each step independently testable.

Tier 1: Gatherers

Collect, extract, classify, structure. Never make strategic decisions.

  • Support Classifier ~$0.02
  • Bug Evidence Collector ~$0.03
  • Feature Context Collector ~$0.03
  • Code Context Mapper ~$0.04
  • Deploy Status Collector ~$0.01
Haiku · GPT-4o-mini · Gemini Flash

Tier 2: Thinkers

Analyze, reason, plan, score, implement. The expensive brain.

  • Support Responder (RAG) ~$0.15
  • Bug Analyst ~$0.30
  • Feature Analyst ~$0.30
  • Coding Agent ~$0.50-1.00
Sonnet · Opus · GPT-4o · Gemini Pro

Tier 3: Communicators

Translate outcomes into empathetic, clear human language.

  • Resolution Notifier ~$0.03
  • Board Summarizer ~$0.01
  • Release Notes Generator ~$0.05
Sonnet · GPT-4o-mini
Contract Architecture

Dual-Layer Ticket Design

Every ticket has a thin Human Layer for board oversight and a deep Agent Layer engineered as a prompt for the next agent in the pipeline.

Human Layer

What the founder sees on the board card. Glanceable, actionable.

title "Login fails on Safari after password reset"
type Bug
priority P1
roiScore 78 / 100
pipelineStage Ready for Implementation
waitingOn agent
humanActionRequired null (agents handle it)

Agent Layer

The full prompt-engineered context bundle consumed by agents via MCP.

context User input, conversation, env, error logs, similar tickets, code context
analysis Root cause, fix tiers (hotfix/proper/refactor), ROI breakdown
guidance Role, objective, steps, success criteria, constraints for next agent
handoffNotes[] Chain of reasoning from every agent that touched this ticket
execution Branch, PR, test results
delivery Deploy status, notification status

Key Risks & Mitigations

The things that could go wrong and how we plan to prevent them.

1

Guidance Template Quality

Templates are prompt engineering at system level. Bad templates = bad agent output regardless of model. Mitigation: Treat as code. Version, test empirically, iterate based on real output.

2

T1 Garbage = T2 Garbage

If evidence collectors extract wrong data, analysts reason about nonsense with high confidence. Mitigation: Strong T1 validation. Cheap to rerun. "Re-gather" button for humans.

3

Cost Explosion

Every agent call = LLM costs. Full bug pipeline: ~$0.50-$2.00. At scale, costs compound. Mitigation: Per-agent daily budgets. Visible per-ticket cost. Cost dashboard.

4

Human-in-the-Loop Fatigue

Too many approval requests drown the founder. Mitigation: Agents proceed autonomously by default. Only flag when confidence < 0.5 or action is high-risk.

5

Pipeline Stalls

A failed agent step = tickets pile up. Mitigation: Per-stage timeouts. Auto-flag "action required." Per-list age tracking surfaces bottlenecks visually.

6

Scope: 11 Components

Risk of building everything at 50% quality. Mitigation: Phase 1+2 delivers a complete intake-to-analysis loop. Ship and validate before Phase 3.

7

Codebase Divergence

FORGE and AIDEN will drift over time. Mitigation: FORGE code in forge-* namespaces. Upstream remote for sync. Monthly merge cadence.

8

Target User Simplicity

Founders understand economics, not kubectl. Mitigation: Business language everywhere. "Deploy latest version" not "Rollout restart deployment."
